Утилита должна уметь не только работать с буфером обмена и текстом, но и отслеживать контекст (какое окно сейчас активно), чтобы шорткаты не конфликтовали в разных программах.

Вот подробный разбор юзкейсов, архитектуры и плана разработки.

### 1. Кейсы использования

**Базовая рутина (Глобальные шорткаты)**

* **Сниппеты текста:** `@@` -> `твой_email@domain.com`, `!тел` -> `+7 (999) ...`.
* **Шаблоны ответов:** Быстрая вставка типовых формулировок для писем, реквизитов или путей к часто используемым сетевым папкам.
* **Управление средой:** Сочетание клавиш для быстрого переключения профилей прокси-серверов или поднятия локальных туннелей прямо из фона.

**Автоматизация в CAD (на примере AutoCAD)**

* **Быстрый вызов кастомных команд:** Ввод `!db` может автоматически отправлять в командную строку вызов твоего LISP-плагина или C#-модуля (например, для отрисовки балок), минуя поиск по ленте.
* **Макросы очистки:** Сочетание клавиш, которое последовательно запускает `_purge`, затем `_audit` и `_qsave` без необходимости писать LISP-скрипт для конкретного чертежа.
* **Вставка типовых узлов:** Шорткат, который вставляет заранее определенный блок с нужным масштабом по координатам мыши.

**Автоматизация в расчетных комплексах (на примере Midas GTS NX)**

* **Комплексные скрипты:** Ввод `#vb` запускает внешний Python-скрипт, который через API или автоматизацию GUI (координаты, клики) генерирует набор Virtual Beams для свайного поля.
* **Автозаполнение координат:** При задании граничных условий или нагрузок часто приходится вбивать одни и те же векторы (например, `0, 0, -1`). Программа может перехватывать шорткат и мгновенно заполнять нужные поля в активном окне.
* **Сборка логов/результатов:** Шорткат, который делает скриншот текущего состояния модели, запрашивает у тебя имя узла и сохраняет картинку с нужным неймингом в папку проекта.

**Разработка и код**

* **Бойлерплейты:** В IDE ввод `!lispdef` разворачивает базовую структуру функции на LISP с комментариями, а `!csplug` — каркас команды для AutoCAD API на C#.

### 2. Архитектура (C#, WinForms + WinAPI)

```text
MacroEngine/
├── core/
│   ├── InputBuffer.cs       # Буфер ввода + символьный детектор триггеров
│   ├── KeyInterceptor.cs    # Глобальный хук (SetWindowsHookEx) + ToUnicodeEx
│   ├── NativeMethods.cs     # P/Invoke: хук, окна, буфер обмена, SendInput
│   ├── TriggerConfig.cs     # JSON-конфиг + FileSystemWatcher (hot-reload)
│   └── WindowContext.cs     # Активное окно (процесс + класс + заголовок) + раскладка
├── modules/
│   └── TextExpander.cs      # Text Expansion (Backspace → Clipboard → Ctrl+V)
├── config/
│   └── triggers.json        # Триггеры
├── ui/
│   ├── SettingsForm.cs      # DataGridView редактор триггеров
│   └── TrayApplicationContext.cs  # Системный трей + координация
└── Program.cs               # [STAThread] точка входа

### 3. План разработки (актуальный)

**Этап 1: Proof of Concept (PoC) ядра** ✅

* ✅ Глобальный хук клавиатуры (`core/KeyInterceptor.cs` — `SetWindowsHookEx`)
* ✅ Буфер ввода: отслеживание последних N нажатых клавиш (`core/InputBuffer.cs`)
* ✅ Text Expansion: Backspace → Clipboard → Ctrl+V (`modules/TextExpander.cs`)
* ✅ Символьное сопоставление триггеров (раскладконезависимо, через `ToUnicodeEx`)

**Этап 2: Конфигурация и контекст окон** ✅

* ✅ `triggers.json` — все триггеры в конфиге, без перекомпиляции
* ✅ Hot-reload через `FileSystemWatcher` (`core/TriggerConfig.cs`)
* ✅ Контекст окна: процесс + класс окна + заголовок (`core/WindowContext.cs`)
* ✅ Контекстные паттерны: `*`, `acad`, `acad,revit`, `*,!browser` (OR + exclusion)
* ✅ Сброс буфера при смене активного окна

**Этап 3: Интерфейс (UI)** ✅ ← только что

* ✅ Системный трей: Start/Stop/Reload/Quit (`ui/TrayApplicationContext.cs`)
* ✅ Редактор триггеров: DataGridView с Add/Delete/Save (`ui/SettingsForm.cs`)
* ✅ Защита от залипания: `IsSuppressed`, корректный Dispose хуков

**Этап 4: Интеграция с рабочим софтом** 🔜

* 🔜 ScriptRunner — запуск внешних скриптов по action=script (`modules/ScriptRunner.cs`)
* 🔜 GuiAutomator — эмуляция кликов мыши для Midas/AutoCAD (`modules/GuiAutomator.cs`)
* 🔜 Система задержек (sleep) для сложных макросов

**Этап 5: Сборка и деплой** 🔜

* 🔜 `dotnet publish` → одиночный exe
* 🔜 Автозагрузка Windows
